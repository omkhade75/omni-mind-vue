## What is OmniMind AI?

**OmniMind AI** is not a chatbot. It is an **Autonomous Enterprise Decision Intelligence Platform** — a fully integrated business operating system built for mall operators, retail chains, and multi-department enterprises.

It replaces the need for separate dashboards for finance, inventory, CRM, billing, HR, and logistics with a **single unified intelligence layer** that:

- Queries your **live PostgreSQL database** in real-time
- Generates **evidence-backed, auditable business decisions**
- Scores every conclusion with a **confidence level**
- Identifies **who** should act, **what** they should do, and **why it's urgent**
- Chains decisions across domains (e.g., low inventory → supplier delay → revenue impact → CEO alert)

> **OmniMind never fabricates data. Every number, every recommendation, every alert traces directly to the database.**

## Core Modules

### 🧠 Decision Intelligence Engine
The heart of OmniMind. Ask any business question in plain English. The engine routes it through a **multi-role reasoning layer** — CEO, CFO, COO, Supply Chain Director, CRM Head, and Risk Analyst — and returns a structured decision report:

```
What happened?          → Evidence-backed observation
Why did it happen?      → Root cause analysis
What is the proof?      → Direct database citations
What happens next?      → Predictive outcome if unaddressed
What action to take?    → Prioritized recommendations
Who should act?         → Role assignment
Urgency level?          → CRITICAL / HIGH / MEDIUM / LOW
Confidence score?       → 0–100% based on data coverage
```

```

Products ───────────► Inventory ──────► InventoryAlerts
         └──────────► BillItems ───────► Bills ──────────► Customers
Suppliers ──────────► PurchaseOrders ──► PurchaseItems
Expenses ───────────► ExpenseCategories
Staff ───────────────► Attendance
Departments ─────────► Utilities
                        Transactions (unified ledger)
                        DecisionLog (AI audit trail)
```

Every AI decision is grounded in these models. Every recommendation cites specific records.

## Security & Access Control

OmniMind implements a **role-based access control (RBAC)** system:

| Role | Access Level |
|---|---|
| **Owner** | Full access — all modules, all financial data, all settings |
| **Admin** | Operational access — most modules, no root settings |
| **Manager** | Floor-level access — inventory, billing, customers |

- Protected route guards on every dashboard route
- Server-side session validation on all API endpoints
- Sensitive financial data fenced behind authentication layer
- Audit log of all critical operations

## Getting Started

### Prerequisites

```bash
Node.js >= 18
PostgreSQL >= 15
Bun (recommended) or npm
```

### 1. Clone the repository

```bash
git clone https://github.com/your-username/omni-mind-ai.git
cd omni-mind-ai
```

### 2. Install dependencies

```bash
bun install
# or
npm install
```

### 3. Configure environment variables

```bash
cp .env.example .env
```

Edit `.env` with your credentials:

```env
DATABASE_URL="postgresql://user:password@localhost:5432/omnimind"
GEMINI_API_KEY="your-gemini-api-key"
SESSION_SECRET="your-session-secret"
```

### 4. Set up the database

```bash
# Push schema to your database
npx prisma db push

# (Optional) Seed with sample data
npx prisma db seed
```

### 5. Start development server

```bash
bun run dev
# or
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) and log in to the dashboard.
